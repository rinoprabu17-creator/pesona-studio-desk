# Owner Sign-Off Templates

Owner sign-off is a business and operational decision. It is not an automation trigger and does not run deployment, backup, restore, scheduler, publisher, upload, social API, OpenAI, queue, or worker behavior.

## Technician Evidence Accepted

```text
date:
release_tag:
branch:
decision: technician evidence accepted
evidence_reviewed:
risks_accepted:
next_action:
owner_name/sign_off:
```

## Sign-Off Notes

- Sign-off should name the reviewed evidence, not just the desired outcome.
- Sign-off should record accepted risks explicitly.
- Sign-off should not include credentials.
- Sign-off does not replace backups, restore dry-run evidence, or route smoke evidence.
